Software Developer,
Phoenix Interactive Design Inc,
London Ontario
During my second internship at Phoenix Interactive Design Inc, I worked as a software developer working on Phoenix's flagship product VistaATM, which is a software solution designed for Automated Teller Machines. My work on VistaATM involved working on developing and implementing new solutions as well as providing software support. At Phoenix I got the opportunity to work on some cutting edge technology in terms of both software and ATM hardware. I felt lucky to be a part of development of next generation solutions for ATM machines. 

My work at Phoenix involved coding in C++ coupled with XML configurations along with some tools based on .NET framework. I was a part of a team that was working on VistaATM's new software release. Being a part of this team, I got the opportunity to work with some of the senior most and smartest people in the company, got exposed to scrum based software development practice as well as got experience coordinating with overseas resources working in Australia, UK and India.


At Phoenix, I was lucky to be assigned to a major release project as it gave me a chance to work on the company's core product and straight-away dive into the code. Because of the large codebase and vast extent of concepts involved in VistaATM development, starting work on Vista involved a steep learning curve. Although challenging to begin with, it became subsequently fun through my increased involvement and help provided by senior colleagues. 

Here are the things I accomplished during my internship at Phoenix : 
  1. VistaATM base Class - I was assigned the job to write VistaATM base class to prevent timeout issues for waiting threads while performing a transaction. Although code for the class was fairly simple, I was introduced to the practical implementation of the concept of multithreading. This class involved a lot of concepts used in Microsoft windows.
  2. VistaATM software Defects - The project being in its wrap up phase, the team was faced by a lot of tickets with software bugs. Some of the tickets were assigned to me. Working on software defects requires an overall understanding of the application. This makes it very challenging to solve software defects. However with the help of my mentor, team members and tech leads, I was able to find solutions and implement the solutions for the defects. Some of the major tickets I solved were
           (i) Free cash on startup in case of power failure  - This bug lead to delivering free cash to the next customer if there was a power outage              as the previous customer was depositing cash. This was a show-stopper bug as the product could not be released without this bug.                Solution to this involved C++ and XML changes to make sure that any wash lying in the machine on startup is not dispensed to the                    next customer.
          (ii)  Incorrect message Displays -  This bug led to display of incorrect message on the screen on wrong card entry. This was associated                   with a missing flowlink in the XML configuration.
  3. Unit Tests - Unit tests are a very vital part of software development. I was given the role to write some new unit tests as well as fix some old ones which were failing due to changes done the respective functions. The unit tests were implemented using Google's Gmock and GTest Framework.
  4. LogMerge MediaMovement Utility - I took the role to create a utility that would extract data from log files and represent it in an easy to read form which would be used by the developers as well as QA to debug and find bugs. This utility was implemented in C# and was an add-on to an existing one.


Overall it was a great experience working at a fast paced environment at Phoenix. The atmosphere at Phoenix was very lively especially the festival celebrations, Halloween, Thanksgiving and Christmas.
